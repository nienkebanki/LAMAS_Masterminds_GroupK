Game

For this research, we use a simplified version of Master(s)Mind(s). Master(s)Mind(s) is a two-player game in which players $i$ and $j$ each secretly choose a code. The setting follows the game introduced by Grachten and Druiven and discussed by Kooi (2000) and Verbrugge & Mol (2008), but our simplified variant is adapted to keep the state space small and to match the modelling choices used later in the report. Concretely, the code length is fixed to 3, the available colours are restricted to five (red, blue, yellow, green, pink), and guesses and codes are required to contain no repeated colours. These restrictions reduce the number of possible codes and make it feasible to compute updates and compare strategies systematically.

red blue yellow green pink W white B black

For this research we chose a simplification where a code is a sequence of length 3 (in the original game it is code length 4) drawn from the five colours above, without repetition (in the original game repetition is allowed). Thus each player has $5 \cdot 4 \cdot 3 = 60$ possible secret codes. At the start of the game, each player knows their own secret code. Before any moves are played, all 60 opponent codes are considered possible.

Feedback

Each round, the active player (starting with $i$) makes a legal guess $g$ of length 3. Feedback is computed using the usual Mastermind comparison rule between a guess and a secret code. First, the number of black pins is the number of positions where the guess matches the secret code exactly (same colour in the same position). Second, the number of white pins is the number of colours that occur in both the guess and the secret code, but are not already counted as black pins. Since repetitions are not allowed, the white-pin count is simply the size of the colour-intersection minus the black-pin count.

The Master(s)Mind(s) twist is that after a guess $g$, the publicly observed outcome consists of two such feedback values:

  1. The feedback comparing $g$ with $j$'s secret code.
  2. The feedback comparing $g$ with $i$'s secret code.

Thus the same public outcome can help the active player learn about the opponent while also giving the opponent information about the active player's code.

Game rules

The game proceeds as follows:

  1. At the start of the game, players $i$ and $j$ each select a random secret code from the 60 legal codes.
  2. On a turn, the active player chooses a legal guess $g$ (length 3, no repeated colours).
  3. Two feedback values are computed by the agents themselves (against the opponent's code and against the active player's own code) according to the Mastermind rule above, and publicly reveals the black pins, white pins or no pins.
  4. The turn then switches to the other player.
  5. A classical win occurs immediately after a guess if the active player's guess exactly matches the opponent's code (i.e. the feedback against the opponent is $3$ black pins). In this report, however, our evaluation focuses on epistemic success (knowledge-based identification) rather than classical win events, since strategies are defined in terms of information gain and leakage.
  6. In addition, an epistemic win for player $i$ occurs when, based on the public history, $i$ can uniquely determine $j$'s code (i.e. only one opponent code remains consistent with everything observed so far).

We evaluate epistemic win conditions after each full round (i.e. after both players have made one guess and the corresponding public announcements have been applied). Thus, an epistemic win for player $i$ occurs at the end of a round when, from the accumulated public history, $i$ can uniquely determine $j$'s code, i.e. when its remaining candidate set for the opponent is a singleton. Because public announcements are shared, it is possible for both players to reach such a singleton state within the same round; we record this as a tie in the sense of symmetric epistemic success.

Toy example

This example uses a smaller toy setting (length $2$ over $\{r,g,b\}$ without repetition) purely to illustrate turn order and feedback. It is important to note that the colors are read through the perspective of agent $j$. This is because it is otherwise confusing in explain the order of the code when having a double feedback.

In Figure 1, agent $i$ is on the left and agent $j$ on the right, each with a hidden two-colour code which is present to the themselves but not to the other agent. The numbers on top show the turn order: $i$ guesses on odd turns, $j$ on even turns. After each guess, both players get feedback (one vs $i$’s code and one vs $j$’s), where white means “right colour, wrong position” and black means “right colour, right position.”

On turn 1, $i$ guesses (green, blue). Against $i$’s code this gives one white pin, and against $j$’s code it also gives one white pin: in both cases, exactly one of the guessed colours is present but misplaced.

On turn 2, $j$ infers from the single white pin vs $i$ that exactly one of {green, blue} occurs in $i$’s code, and it is in the other position. In the situation shown, that leaves two candidates for $i$: (red, green) or (blue, red). Since (blue, red) is $j$’s own code, $j$ chooses the other option and guesses (red, green), which yields two black pins. Because it is $j$’s turn, $j$ wins.
