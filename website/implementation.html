What we implemented

  • Enumeration of all codes (6 permutations) and worlds (36 pairs).
  • Public-announcement update by filtering worlds inconsistent with $(F_B,F_A)$.
  • Candidate sets $Cand_A(B)$ and $Cand_B(A)$ computed from remaining worlds.
  • Higher-order knowledge check for $K_A(|Cand_B(A)|\ge k)$.
  • Strategies: GreedyGain, MinLeak, Trade-off, HO-Constraint.
  • CLI interaction where $c_B$ is unknown to the user (default mode).

Simplifications

  • 3 colors and length 3 (small state space, easy to inspect, still shows DEL + HO effects).
  • No repetition in codes.
